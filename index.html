Installation

Follow the steps below to setup your SuperUser profile:

  1. Click Here to Download this Script [ Superuser ] as Zip File
  2. Unzip the downloaded package and open the /SuperUser folder to find all the files.
  3. Below is the directory structure:

    • PowerShell/Microsoft.PowerShell_Profile.ps1 - When the Initialization Begins
    • PowerShell/assets//website - Contains all assets//website for the Script
    • PowerShell/data - Contains all of The Configuration Data for the Script
      • PowerShell/data/ascii.json - ASCII Typography Files
      • PowerShell/data/config.json - Configuration Data File
      • PowerShell/data/directories.json - Contains lists of directories & releated info
      • PowerShell/data/files.json - Contains lists of files & releated info
      • PowerShell/data/settings.json – All external libs.
    • PowerShell/docs - This Documentations that you're reading.
    • PowerShell/profile - Contains all of the assets//website referenced
      • PowerShell/profile/auto - Contains Auto-Discover Scripts
      • PowerShell/profile/cmdlets - Small Few Lines Scripts
      • PowerShell/profile/dependency - Profile Dependend Script files
      • PowerShell/profile/functions - Callable Profile's Function Scripts
      • PowerShell/profile/prompt – Personalized SuperUser Prompt.
    • PowerShell/superuser.ps1 - The Configuration File

  4. You will need to move these files to your User's Documents.

    NoteRename the /SuperUser folder to /PowerShell.

    For Windows 11

    C:/Users/<username>/OneDrive/Documents/PowerShell

    For Windows 10

    C:/Users/<username>/Documents/PowerShell

    You Directory Path Should look similar to the above path!

  5. Open Environment Variable Window (Search env from your taskbar & Open it) and
    Add below Key-Value Pairs

    6. Pair #1

    
  Key : superuser 
  Value : C:\Users\<Username>\OneDrive\Documents\PowerShell\superuser.ps1

    Pair #2

    
  Key : superuser_data
  Value : C:\Users\<Username>\OneDrive\Documents\PowerShell\data

    Pair #3

    
  Key : superuser_profile
  Value : C:\Users\<Username>\OneDrive\Documents\PowerShell\profile

    NoteYou need to replace <Username> with your Username

  6. Open The Terminal | Enjoy

1. access the JSON File Content

Read the JSON File & Return the JSON Object[]


        function ReadJson($JsonFile) {
          $json_data = Get-Content -Raw -Path $JsonFile | ConvertFrom-Json
          return $json_data
        }

This Function ReadJson() have one parameter $JsonFile which takes the path of the json file as an argument
and returns the JSON Object which futher can be used to fetch the value of elements of the json file

Fetch Path From Environment-Variable


    $su = $env:superuser_data

Fetch Json Path from the 'Environment Variable -> superuser_data
( C:\Users\<Username>\OneDrive\Documents\PowerShell\data )

JSON Files in the SuperUser's Data Directory 


    $Data = @{

        "dir"      = "$su\directories.json"

    }

HashTable to store the files's path from the $env:superuser_data which can be retrieved later

Save JSON Obect[] in Variable


  $Ascii_JSON = ReadJson($Data.ascii)
  $Files_JSON = ReadJson($Data.files)

Pass the json-file-path to the function & Stores the Obejct[] in the Variable

Returns Full Path


    function ReadJsonPath($JsonPointer) {
      $profile_home = "$home\OneDrive\Documents"
  
      $full_path = "$profile_home\$($JsonPointer)" 
  
      # Write-Host "[ From-JSON ] File Path : $full_path" -ForegroundColor Cyan

Read the Short-Path from the JSON file and Returns the Full-Path

      $ascii_terminal = ReadJsonPath($Ascii_JSON.ascii.text.file[1].path)

How to fetch Value from the JSON file

Content of the JSON File $su/ascii.json


      {
        "info" :
        {
            "title" : "ascii",
            "file"  : "ascii.json",
            "path"  : "PowerShell\\data\\ascii",
            "ascii" : "PowerShell\\data\\ascii\\text\\powershell_basic.ascii",
            "desc"  : "contains sub-directories from the powershell directory"
        },
        "ascii" :
        {
            "script" :
            {
                "info":
                {
                    "title" : "ascii_script",
                    "file"  : "",
                    "path"  : "PowerShell\\data\\ascii\\script",
                    "ascii" : "",
                    "desc"  : ""
    
                },
                "file" : 
                [
                    { "path" : "PowerShell\\data\\ascii\\script\\neofetch.ps1"},
                    { "path" : "PowerShell\\data\\ascii\\script\\neofetch_windows.ps1"}
                ]
    
            },
            "text" :
            {
                "info":
                {
                    "title" : "ascii_text",
                    "file"  : "",
                    "path"  : "PowerShell\\data\\ascii\\text",
                    "ascii" : "",
                    "desc"  : ""
                },
                "file" :
                [
                    { "path" : "PowerShell\\data\\ascii\\text\\powershell_basic.ascii"},
                    { "path" : "PowerShell\\data\\ascii\\text\\superuser.ascii"},
                    { "path" : "PowerShell\\data\\ascii\\text\\theterminal.ascii"}
                ]
            }
    
        }
    }

In order to read & fetch the value. We need to follow the FAMILY-TREE Concept
At first there is the Grandfather who have their Children & Sometimes the Children have their own kids GrandChildren 


      ascii.json {}
          |
          |──────info {}
          |──────ascii {}
          |       ├────.\script {}
          |       └────.\text {}  
                         ├───.\info {}
                         └───.\file []
                                ├────.\path [0] -> powershell_basic.ascii
                                ├────.\path [1] -> superuser.ascii (value we're interested in)
                                └────.\path [2] -> theterminal.ascii

We are interested in below Key-Value Pair. More Specifically the the second one 


      "file" :
      [
          { "path" : "PowerShell\\data\\ascii\\text\\powershell_basic.ascii"},
          { "path" : "PowerShell\\data\\ascii\\text\\superuser.ascii"},
          { "path" : "PowerShell\\data\\ascii\\text\\theterminal.ascii"}
      ]

Lets Access the JSON Using Pointers
We have to point to the value we're interested in!

  • At first We have the variable which contins the directory where json-file is located $su = $env:superuser_data (value fetched from Environment-Variable)
  • Then comes the variable which contains the JSON Object which is $Ascii_JSON = Get-Content -Raw -Path "$su/ascii.json" | ConvertFrom-Json
  • Then we need to point to the Key whose value we're interested in
  • In this case we are interested in retrieving the value superuser.ascii
  • Notice the {} - Curly Brackets | [ ] - Square Brackets

        ascii.json {}  
            |
            |──────info {}  -> $Ascii_JSON.info (this will return as object - as there are children keys within this)
            |       ├────.\title -> $Ascii_JSON.info.title 
            |       └────.\file  -> $Ascii_JSON.info.file
            |──────ascii {}
            |       ├────.\script {}
            |       └────.\text {}  
                           ├───.\info {}
                           └───.\file []
                                  ├────.\path [0] -> $Ascii_JSON.ascii.text.file[0].path
                                  ├────.\path [1] -> $Ascii_JSON.ascii.text.file[1].path
                                  └────.\path [2] -> $Ascii_JSON.ascii.text.file[2].path
  • Whenever the key we are accessing is inside the { } - we simply access it using ( . ) - just dot and the Key that Whose Value we want.
  • And Whenever the key we are accessing is inside the [ ] - we simply access it using ([ $index_no ]) - where index_no is an integer
  • In an Array the index_no starts from zero
